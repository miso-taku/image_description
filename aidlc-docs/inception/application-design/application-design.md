# アプリケーション設計（統合）

本書は components.md、component-methods.md、services.md、component-dependency.md を統合した高レベル設計である。

## 概要

`image-description-app` は、Next.js/ReactフロントエンドとFastAPI/PydanticAIバックエンドで構成するローカル動作の画像説明アプリケーションである。ユーザーが選択した画像を、詳細度指定に応じてOpenAIのビジョン対応モデルで日本語説明に変換する。

## アーキテクチャ方針

- 関心の分離: API層、設定、検証、AIサービス、スキーマ、例外を分離。
- 多層防御: クライアント検証とサーバー検証を独立実装。
- 安全なエラー: 失敗は必ずドメイン例外を経てユーザー安全なJSONへ整形。
- プライバシー: 画像と説明を永続化せず、機密情報をログに残さない。
- 設定可能性: モデル、タイムアウト、CORS、サイズ上限、レート制限を環境変数化。

## コンポーネント一覧

- バックエンド: API層、設定、画像検証、AI説明サービス、スキーマ、例外・ロギング、セキュリティ。
- フロントエンド: ページ、アップロード、結果、APIクライアント、クライアント検証。

## 主要サービス

- 画像説明サービス（オーケストレーション）
- 検証サービス（純粋ロジック）

## 悪用ケースの考慮（SECURITY-11）

- 巨大ファイルやゼロバイトファイルによる資源枯渇 -> サイズ・空チェックで拒否。
- 偽装拡張子・偽装Content-Type -> ファイルシグネチャ検証。
- 連続リクエストによる乱用 -> ローカル向けレート制限。
- プロンプトインジェクション目的の画像 -> システムプロンプトで断定回避と役割固定。

## 設計判断

- 単一ユニットとして扱い、Units Generationは不要（実行計画に記載）。
- OpenAI連携はPydanticAI Agentに集約し、テスト時はモデルをスタブ化可能にする。
