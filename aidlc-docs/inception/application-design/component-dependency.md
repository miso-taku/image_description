# コンポーネント依存関係

## 依存マトリクス（バックエンド）

| コンポーネント        | 依存先                                         |
| --------------------- | ---------------------------------------------- |
| API層 (C-B1)          | 設定, 画像検証, AI説明サービス, スキーマ, 例外 |
| 設定 (C-B2)           | Pydantic Settings                              |
| 画像検証 (C-B3)       | なし                                           |
| AI説明サービス (C-B4) | 設定, スキーマ, PydanticAI                     |
| スキーマ (C-B5)       | Pydantic                                       |
| 例外・ロギング (C-B6) | FastAPI, logging                               |
| セキュリティ (C-B7)   | 設定, FastAPI                                  |

## 依存マトリクス（フロントエンド）

| コンポーネント          | 依存先                       |
| ----------------------- | ---------------------------- |
| ページ (C-F1)           | アップロード, 結果, APIクライアント |
| アップロード (C-F2)     | クライアント検証             |
| 結果 (C-F3)             | なし                         |
| APIクライアント (C-F4)  | なし                         |
| クライアント検証 (C-F5) | なし                         |

## 通信パターン

```text
[Browser: Next.js/React]
        |  multipart/form-data (image, detail)
        v
[FastAPI API層] --> [検証サービス] --> [AI説明サービス] --> [PydanticAI] --> [OpenAI API]
        ^                                                                      |
        |------------------- JSON (description | error) -----------------------|
```

## データフロー

1. ユーザーが画像と詳細度を選択。
2. フロントエンドがクライアント検証後にバックエンドへ送信。
3. バックエンドがサーバー検証後、AI説明サービスを実行。
4. PydanticAI経由でOpenAIが説明を生成。
5. バックエンドがJSONを返し、フロントエンドが表示。
6. 画像とレスポンスは永続化されず処理後に破棄。
