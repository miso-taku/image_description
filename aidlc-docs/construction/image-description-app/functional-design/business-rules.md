# ビジネスルール

## BR-01 入力形式

- 許可MIME: `image/jpeg`, `image/png`, `image/webp`。
- ファイルシグネチャで実形式を確認する（拡張子・Content-Typeのみを信用しない）。
  - JPEG: `FF D8 FF`
  - PNG: `89 50 4E 47 0D 0A 1A 0A`
  - WebP: 先頭 `RIFF` かつ 8バイト目以降に `WEBP`
- 宣言Content-Typeと実シグネチャが矛盾する場合は拒否する。

## BR-02 サイズ

- 最大サイズは10 MB（既定、環境変数で変更可能）。
- 0バイトは拒否する。
- 検証はOpenAI呼び出し前に行う。

## BR-03 詳細度とプロンプト

- `brief`: 1〜2文で主要被写体と状況。
- `standard`: 被写体、背景、構図、色、動作を簡潔な段落で。
- `detailed`: 観察要素、位置関係、雰囲気、文字情報を複数段落または箇条書きで。
- 既定は `standard`。
- 不明な値は拒否する（列挙で検証）。

## BR-04 生成方針（安全性）

- 出力言語は日本語。
- 画像から読み取れない情報を事実として断定しない。
- 人物の身元、センシティブ属性、正確な所在地を根拠なく断定しない。
- 読み取れない文字は「判読できない」と明示できる。

## BR-05 エラー分類

| コード                  | 契機                       | HTTP |
| ----------------------- | -------------------------- | ---- |
| `invalid_image_type`    | 未対応MIME/シグネチャ不一致 | 400  |
| `empty_image`           | 0バイト                    | 400  |
| `image_too_large`       | サイズ超過                 | 413  |
| `invalid_detail`        | 不正な詳細度               | 400  |
| `configuration_error`   | APIキー未設定など          | 503  |
| `authentication_error`  | OpenAI認証失敗             | 502  |
| `rate_limited`          | OpenAIレート制限           | 429  |
| `upstream_timeout`      | OpenAIタイムアウト         | 504  |
| `upstream_error`        | OpenAIその他失敗           | 502  |
| `rate_limited_local`    | ローカルレート制限         | 429  |
| `internal_error`        | 予期しない例外             | 500  |

## テスト対象プロパティ (PBT-01/03/07)

- **BR-02 サイズ不変条件**: `max_bytes` 以下の非空データは常に通過、超過は常に拒否。
- **BR-01 MIME判定**: 有効なシグネチャ先頭を持つデータは対応MIMEへ判定される。
- **BR-03 詳細度**: 定義済み列挙値のみ受理し、それ以外は必ず拒否。
- 生成には制約付きジェネレーター（有効・無効なバイト先頭、境界サイズ）を使用する (PBT-07)。
