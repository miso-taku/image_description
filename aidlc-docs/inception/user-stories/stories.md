# ユーザーストーリー

INVEST原則に従い、初期リリース範囲のストーリーを定義する。

## US-01 画像を選択する

**As** みなと **I want** 端末から画像を1枚選択したい **so that** 内容説明を依頼できる。

**受け入れ基準**

- クリックとドラッグ＆ドロップの両方で選択できる。
- プレビュー、ファイル名、サイズが表示される。
- 新しい選択で以前の画像が置き換わる。

## US-02 入力形式を検証する

**As** そら **I want** 未対応形式や大きすぎるファイルを弾いてほしい **so that** 無駄なAPI呼び出しを避けられる。

**受け入れ基準**

- JPEG、PNG、WebPのみ許可する。
- 10 MB超と空ファイルを拒否する。
- クライアントとサーバーの双方で検証する。
- エラーは日本語で表示される。

## US-03 説明の詳しさを選ぶ

**As** みなと **I want** 説明の詳しさを選びたい **so that** 目的に合った説明量を得られる。

**受け入れ基準**

- 短い、標準、詳細から選べる。
- 既定は標準。
- 選択がバックエンドのプロンプトへ反映される。

## US-04 画像の説明を生成する

**As** みなと **I want** 画像の説明を日本語で生成したい **so that** 内容を理解できる。

**受け入れ基準**

- 画像と詳細度がバックエンドへ送信される。
- PydanticAI経由でOpenAIモデルが呼び出される。
- 日本語の説明が返る。
- 画像がOpenAIへ送信されることが送信前に明示される。
- 処理中は状態が示され重複送信が防止される。

## US-05 結果を確認しコピーする

**As** みなと **I want** 生成結果を読みコピーしたい **so that** ほかの場所で使える。

**受け入れ基準**

- 説明が読みやすく表示される。
- コピー操作と結果通知がある。
- 結果更新がスクリーンリーダーへ通知される。

## US-06 エラーを理解する

**As** そら **I want** 失敗理由を安全に把握したい **so that** 適切に対処できる。

**受け入れ基準**

- 検証、設定不足、認証、レート制限、タイムアウト、障害を区別する。
- 内部情報やAPIキーを露出しない。
- 再試行可能性を伝える。

## US-07 プライバシーを守る

**As** みなと **I want** 画像や説明が保存されない安心がほしい **so that** 安全に利用できる。

**受け入れ基準**

- アプリは画像と説明を永続保存しない。
- ログに画像内容と説明本文を出力しない。

## US-08 アクセシブルに操作する

**As** みなと **I want** キーボードと支援技術で操作したい **so that** 誰でも使える。

**受け入れ基準**

- 全操作がキーボードで可能。
- フォーカスとラベルが明確。
- 状態を色以外でも伝える。

## US-09 ローカルで起動する

**As** そら **I want** バックエンドとフロントエンドを簡単に起動したい **so that** すぐ試せる。

**受け入れ基準**

- uvでバックエンドを起動できる。
- npmでフロントエンドを起動できる。
- READMEに手順とAPIキー設定が記載される。
